“In programming, simplicity and clarity are not a dispensable luxury, but a crucial matter that decides between success and failure.” - E. Dijkstra

  • Build rules using annotated POJOs.

        @Rule
    public class MessageRule {
        @Given("msg")
        String msg;

        @Then
        public void thenMsg() {
            System.out.println("A message for you: " + msg);
        }
    }

  • Or build rules using the Java DSL.

        RuleBookBuilder ruleBook = RuleBookBuilder.create()
      .addRule(rule -> rule
        .when(f -> true) //not required; shown only for illustrative purposes
        .then(f -> f.setValue("msg", "RuleBook is the best!")))
      .addRule(rule -> rule
        .using("msg")
        .then(System.out::println))
      .build();

  • Run a group of rules you created.

        ruleBook.run(facts);

    Or run one constructed from a package containing rules.

        RuleBook ruleBookFromPkg = new RuleBookRunner("com.example.myrulebook");
    ruleBookFromPkg.run(facts);

  • Oh, and it works with Spring too.

        @Configuration
    public class SpringConfig {
        ...

        @Bean
        public RuleBook springRuleBook() {
            return new RuleBookRunner("com.example.myrulebook");
        }
    }

Meet RuleBook

RuleBook provides a simple, but powerful and flexible rules abstraction that has an incredibly short learning curve. With RuleBook, there’s no guesswork or unpredictable behavior, which means developers can spend more time creating useful functionality. And unlike many other rules engines, RuleBook can be used to drive workflows, where the ordering of rules is important. So, start simplifying your life and streamlining your code. Start using RuleBook today!
